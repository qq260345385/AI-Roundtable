# Frontend Manual Checklist

这份清单用于 internal alpha / pre-release 阶段的前端手动验收，不是正式发布说明。

## 1. Mock 模式正常会议流程

- 设置 `AI_ROUNDTABLE_MODE=mock`，或不设置环境变量。
- 打开首页后，应能明确看到当前是 Mock 模式。
- Mock 文案应说明使用模拟模型、不需要 API key、不代表真实 API 已配置。
- 参会模型应显示 `Mock / 无需 API`。
- 输入议题后点击“开始圆桌会议”，应显示独立观点、自由回应和共识整理。

## 2. Real 模式无 provider 配置

- 设置 `AI_ROUNDTABLE_MODE=real`，不配置可用 provider。
- 首页应提示需要检查 `.env.local` 中的 provider 配置。
- 开始按钮应禁用，并显示“暂无可用模型”。
- 参会模型列表应说明当前 real 模式下没有可用于会议的模型。
- 页面不应显示 API key、Authorization 或 Bearer。

## 3. Real 模式 provider 检测失败

- 配置完整 provider，但让 `{BASE_URL}/models` 不可达或不兼容。
- provider 状态应显示“检测失败 / 未验证”，不能显示为“已连接”。
- 页面应说明该状态不代表连接已验证成功。
- 如果当前策略仍允许参与会议，UI 也应保持“未验证”语义。

## 4. Real 模式 MODEL 不存在

- 配置完整 provider，但填写一个不存在的 `MODEL`。
- 未启用模型区域应显示“模型未找到”。
- 如果接口返回 `detectedModels`，页面只应展示前几个模型，避免列表过长。
- 建议文案应提示检查 MODEL 或改用检测到的模型。

## 5. Real 模式部分 provider 调用失败

- 准备至少两个 provider，其中一个调用会失败。
- 会议不应被显示为整场失败。
- 页面应保留成功模型的发言。
- 失败提示区域应说明“会议已保留其他模型的成功发言”。
- “部分模型调用失败”区域应显示 provider、model、中文阶段、简短原因和建议。
- Markdown 中应包含“模型调用失败记录”。

## 6. Real 模式全部 provider 调用失败

- 让所有可参与 provider 调用失败。
- 页面应走整场错误状态，并显示简洁错误。
- 错误信息不应包含 API key、Authorization、Bearer 或完整第三方错误体。

## 7. Markdown 复制

- 会议成功后才应显示“复制 Markdown”按钮。
- 点击后应出现短暂的复制成功反馈。
- 浏览器拒绝剪贴板权限时，应显示简洁失败反馈。
- 反馈不应长期占据页面。

## 8. 资料文件导入

- 启用“资料文件”后，未选择可解析文件时开始按钮应禁用。
- `.txt`、`.md`、`.csv`、`.json` 文件应能生成资料卡片。
- `.pdf`、`.docx`、`.xlsx`、`.pptx` 文件应能在本地解析接口提取文本并生成资料卡片。
- 资料卡片应显示预览编号、来源、摘要预览、字符数，以及截断或解析质量提示。
- 如果任一资料有提示，开始按钮附近应显示“建议检查资料预览”的轻量提示，但不应阻止开会。
- 扫描版 PDF、图片和旧版 Office 格式无法解析时，应显示简洁失败提示。
- 资料文件最多保留 5 个，超出的文件不应进入会议资料包。

## 9. 敏感信息检查

- 首页、模型列表、失败记录、Markdown 复制结果都不应显示真实 API key。
- 页面不应显示 Authorization header。
- 页面不应显示 Bearer token。
- 页面不应显示完整第三方错误体、内部调试堆栈或本地调试日志。
- 示例内容和截图提交前应再次脱敏。
